# Project Development Proposal

#### Context and purpose

Within the internship scope, the **Mini E-commerce DevOps Platform** project has completed its foundation phase: local development environment, AWS infrastructure managed with Terraform, workload deployment on Amazon EKS, CI/CD automation with GitHub Actions, state synchronization via GitOps (Argo CD), and baseline secret management and system monitoring.

This proposal presents the **next development direction**, aiming to evolve the project from a DevOps demonstration model toward a more production-like operations model while still controlling cost, deployment scope, and technical complexity — suitable for a personal account context and internship report evaluation goals.

#### Scope of application

The proposals in this section **do not replace the entire existing architecture** but add capability layers for availability, security, safe deployment, system observability, and cost governance. Each item can be implemented in phases, with technical evidence for evaluation and reporting.
