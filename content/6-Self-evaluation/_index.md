### Detailed comments

#### Strengths

* **Ability to master container packaging and orchestration technologies:** Completed the entire 12-week worklog in accordance with the AWS Study Group roadmap. I have mastered the Dockerfile writing process to package a variety of applications (from Frontend, API to Database), and gained hands-on experience operating, monitoring, and managing container lifecycles on AWS advanced service environments including Lightsail Container, Amazon ECS, and Amazon EKS.
* **Architectural migration and infrastructure optimization mindset:** Successfully absorbed and applied the mindset of decomposing large, bulky monolithic systems (Monolith) into flexible microservices architectures (Microservices). I have the ability to analyze services based on bounded contexts, establish fault-tolerance mechanisms, configure replica for Pods, handle automatic load balancing, and optimize operational performance for workloads on serverless AWS Fargate.
* **Building automated CI/CD and source code management systems:** Successfully established pipelines to automate continuous integration and deployment workflows using GitHub Actions. The process includes automated source code checking, building Docker images, scanning basic security vulnerabilities, synchronizing source code via Git/SourceTree, pushing products to Amazon ECR repositories, and automatically updating deployment manifests configuration files safely and consistently.
* **Exploring large-scale data processing and analytics solutions:** Possess practical understanding of building data lakes (Data Lake) on Amazon S3, knowing how to use AWS Glue Crawler to automatically index metadata into the Glue Data Catalog. I have also proficiently practiced optimizing query execution via SQL language through Amazon Athena and building visualization charts and dashboards on Amazon QuickSight.
* **Active contribution to the community and technical writing skills:** Participated fully and dynamically in major FCAJ community events (including Community Days and GameDay). I have finished translating and editing multiple technical deep-dive articles on the official AWS blog, thereby greatly enhancing my ability to read and understand technical materials in English as well as express technology topics in Vietnamese.

#### Limitations

* **System debugging speed in advanced networking and security configurations:** In the early stage of approaching complex networking configurations and multi-layered security policies (such as IAM Policies, VPC Peering, Security Groups, or Kubernetes YAML manifests configurations), I was still confused and spent a lot of time searching logs, locating, and resolving the root causes of arising issues.
* **Balancing application operational performance and budget optimization:** When experimentally deploying large service clusters or running real-time data flows, I often prioritized ensuring smooth, stable, and highly available system operation, sometimes without truly optimizing hardware capacity or configuring auto-scaling resources from the start to save costs for the cloud account.
* **Thoroughness in resource control and cleanup:** Due to the dense and continuous schedule of hands-on practice labs, internal time management was sometimes not scientific enough. This led to delayed teardowns of test services (such as EKS clusters, Kinesis connection endpoints, or Athena partition folders), which were not cleaned up immediately after the practice sessions.

#### Needs improvement

* **Enhance independent troubleshooting mindset and skills:** Practice approaching errors systematically by reading deep into logs (system logs/event logs) on CloudWatch and kubectl. Prioritize carefully studying AWS and Kubernetes official documentation to find optimal solutions instead of applying temporary workarounds.
* **Improve resource management mindset under FinOps model:** Focus on cultivating cloud budget control knowledge. I will build and strictly follow a checklist for cleaning up infrastructure after each practice session, and learn how to configure automated budget alerts (AWS Budgets) to manage lab resources smartly and cost-effectively.
* **Upgrade presentation, progress reporting, and communication skills:** Actively participate in deep group discussions, practice presenting infrastructure architecture solutions and progress reports concisely and focused on the key questions and experimental results.
* **Deepen expertise and conquer industry certifications:** Maximize the practical knowledge and experience accumulated from labs and FCAJ GameDay to continue the advanced self-study path. Set clear learning goals to prepare for and pass industry standard professional certifications, starting with AWS Certified Cloud Practitioner and progressing toward AWS Certified Solutions Architect – Associate.
